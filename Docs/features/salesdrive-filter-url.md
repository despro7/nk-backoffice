# SalesDrive — Filter state in URL hash

Коротко: сторінка вивантаження SalesDrive тепер серіалізує поточний стан фільтрів у `window.location.hash` і відновлює його при завантаженні.

Куди дивитись:
- Код реалізації: `client/components/SalesDriveOrdersTable.tsx`

Правила серіалізації:
- `search` — текст пошуку (додається тільки коли непорожній).
- `searchCategory` — додається тільки якщо відрізняється від дефолтного `orderNumber`.
- `channels` — список id каналів, розділених крапкою, наприклад `channels=19.22.24.unknown`.
  - Якщо обрані всі доступні канали — параметр `channels` взагалі не додається (тобто відсутність параметра = всі канали).
- `shipmentFilter` — додається коли відрізняється від `all` (`shipped`, `not_shipped`, `not_shipped_all`, `duplicates`).
- `selectedStatus` — додається коли відрізняється від `all`.
- `dateFrom` / `dateTo` — формат `YYYY-MM-DD`, додаються коли вибрано діапазон.
- `page` / `pageSize` — додаються коли відрізняються від дефолтних (сторінка 1, `pageSize` 20).

Парсинг та відновлення:
- При завантаженні компонент читає `window.location.hash` і намагається застосувати фільтри.
- Якщо канали ще не завантажені з API, парсовані значення тимчасово зберігаються і застосовуються після отримання списку каналів.
- Якщо парсований список каналів не містить жодного валідного id — відбувається fallback на всі доступні канали.

Приклади URL hash:
- Один або декілька каналів: `#channels=19.22.24.unknown&search=abc&page=2`
- Пошук у не-дефолтній категорії: `#search=ivan&searchCategory=name`
- Діапазон дат: `#dateFrom=2026-05-01&dateTo=2026-05-05`

Поради користувачу:
- Щоб поділитися поточним станом фільтрації — скопіюйте URL з браузера (hash включено).
- Якщо URL виглядає довгим, перевірте, чи не обрано занадто багато окремих каналів — у разі «всіх каналів» параметр `channels` буде опущений.

Якщо потрібно — можна додати кнопку «Копіювати лінк» у UI (`client/pages/SalesDriveOrders.tsx`), яка буде копіювати `window.location.href` у буфер обміну.
